# 🧠 One A Day – JavaScript Journal

A personal challenge: **learn one new thing about JavaScript every day** and document it publicly.

This repository is my learning log — a collection of short research notes, code experiments, and reflections as I deepen my understanding of JavaScript fundamentals and internals.

---

## 🎯 Purpose

- Strengthen core JavaScript fundamentals
- Build accurate mental models
- Practice explaining technical concepts clearly
- Create a long-term, searchable knowledge base
- Stay consistent through daily progress

This is not a tutorial or polished blog.  
It’s a record of what I’m actively learning.

---

## 📅 Rules

- One topic per day
- One Markdown file per topic
- Write explanations in my own words
- Include runnable examples where possible
- Be honest about confusion or mistakes

Consistency > perfection.

---

## 📂 Repository Structure

```
/YEAR-MONTH
  01-topic-name.md
  02-topic-name.md
  03-topic-name.md
```

Each month has its own folder.  
Each file represents one day of focused learning.

---

## 📝 Entry Format

Each entry typically includes:

- What the concept is
- Why it matters
- Code examples
- Edge cases or gotchas
- What surprised me
- Further reading
- (Optional) Revisited section later
